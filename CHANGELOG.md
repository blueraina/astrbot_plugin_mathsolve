# 更新日志

## v1.13.3 - 2026-07-06

- 修复普通图文答疑接管数学图片题时可能丢失图片输入的问题。
- 普通答疑与 `/spdf` 的图片提取改为复用 `/pdf` 的稳定快照逻辑，并在插件接管后主动把当前图或引用缓存图写回模型请求。

## v1.13.2 - 2026-07-03

- 修复手动生成答疑报告只统计上次手动触发后记录的问题，改为按当前时间向前统计 24 小时答疑记录。
- 优化日报 LaTeX 排版：专题标题、典型题标题和“核心结论/关键推导/详细过程”等标签会使用 LaTeX 加粗。

## v1.13.1 - 2026-07-03

- 优化每日答疑报告结构：去掉今日概览、串联型专题、并联型专题和今日结论卡片。
- 强化专题讲义和典型题精讲的生成要求，要求展开关键推导并允许使用行间公式。
- 修复日报中 Markdown `**加粗**` 被原样渲染为星号的问题，并让典型题按题目拆成独立卡片。

## v1.13.0 - 2026-07-03

- 将每日答疑报告功能作为正式发布版本推出，并同步 GitHub Release。
- 版本号从 `v1.12.0` 提升到 `v1.13.0`，便于 AstrBot 插件页提示更新。

## v1.12.0 - 2026-07-02

- 新增每日答疑报告：可归档完整图文答疑、`/pdf` 和 `/spdf` 记录，并生成结构化 LaTeX 讲义 PDF。
- 新增 `/答疑报告生成` 手动命令，用于按当前会话的手动区间立即生成并发送测试报告。
- 新增自动日报调度：支持提前开始整理生成，到设定时间自动发送；WebUI 只保留启用、时间、模型、目标会话和篇幅等必要选项。
- `/pdf`、`/spdf` 成功发送后会保存对应 TeX 源码，供日报整理使用。

## v1.11.1 - 2026-06-29

- 修复 Markdown 图片回答中不同颜色讲义卡片的左侧强调线不一致问题，现在题目卡片、定理/提示等引用卡片都会显示对应颜色的左侧粗线。
- 修复证明结尾 `\hfill\blacksquare` 在图片回答中被当作普通文本显示的问题，渲染前会自动转换为右对齐的 MathJax 黑色方块。
